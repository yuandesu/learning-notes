方案 A：Forwarder（間接路徑）

你的 Lambda 函數
console.log() / print()
↓ 自動寫入
CloudWatch Logs
Log Group: /aws/lambda/你的函數
↓ Subscription Filter 觸發
Datadog Forwarder Lambda
獨立的 Lambda 函數
↓ HTTPS
Datadog 後端

經過 CloudWatch → 有 CloudWatch 費用 → 每個 log group 要設 subscription filter

方案 B：Extension（直接路徑）

你的 Lambda 函數
+ Datadog Extension（同一個執行環境）
透過 Lambda Layer 安裝
↓ HTTPS 直接送
Datadog 後端

不經過 CloudWatch → 省 CloudWatch 費用 → 不用設 subscription filter

差異總覽

比較項目ForwarderExtension
本質獨立的 Lambda 函數Lambda Layer
資料路徑函數→CW→Forwarder→DD函數→Extension→DD
經過 CloudWatch？是否
設定每個 log group 要設 filter加一個 Layer
CloudWatch 費用有省掉
Cold start 影響無有一點
非 Lambda log可以（S3, RDS 等）不行
新功能不再更新持續更新
⚠️ 什麼時候還需要 Forwarder？Extension 只處理 Lambda 的遙測。如果你要把非 Lambda 的 AWS 服務 log（S3 access log、RDS log、VPC Flow Log 等）送到 Datadog，還是需要 Forwarder 或 Kinesis Firehose。
