核心概念

EC2 = 在 AWS 實體伺服器上被 Nitro Hypervisor 虛擬化出來的 虛擬機。你擁有完整 OS 控制權。

⚙️

Agent 部署方式

Agent 以 ECS Daemon Scheduling Strategy 部署為容器，自動在每台 EC2 上跑恰好一個。

💰

APM 計費

以 APM Host（= EC2 實例） 為單位。3 台 EC2 跑 50 個容器 → 只算 3 個 APM Host。

✅

官方推薦

ECS on EC2 環境的 首選部署方式，容器探索整合最完整，升級只需更新 image tag。

核心差異

沒有「你的 EC2」。AWS 用 Firecracker microVM 為每個 Task 建立硬體級隔離沙箱。你無法 SSH，也看不到底層 OS。

🔥

Firecracker 是什麼？

AWS 開源的 microVM 引擎，比傳統 VM 更輕（啟動 <125ms），但保有 kernel 級隔離。Lambda 也用它。

💰

APM 計費

以 APM Fargate Task 為單位。10 個 Task 各 1 個 Sidecar → 10 個計費單位。

📐

為什麼必須 Sidecar？

沒有共享主機，不可能 Daemon 模式。Agent 只能放在 Task 內部作為 Sidecar。
